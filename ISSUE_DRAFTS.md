# 이슈 초안

GitHub Issues에 등록할 이슈 초안입니다.

## 1. 실제 환자 데이터 사용 전 Supabase RLS 정책 강화 필요

### 문제

현재 `supabase-schema.sql`의 RLS 정책은 테스트를 쉽게 하기 위한 데모용 설정입니다. anon role이 `public.patients` 테이블의 모든 데이터를 조회, 저장, 수정, 삭제할 수 있습니다.

### 위험

Supabase 프로젝트가 외부에서 접근 가능한 상태라면, public anon key를 가진 사용자가 환자 데이터를 조회하거나 변경할 수 있습니다. 실제 환자 개인정보를 다루는 환경에서는 사용할 수 없는 권한 설정입니다.

### 개선 방향

- 로그인 기능을 붙인 뒤 인증된 사용자만 환자 데이터에 접근하게 합니다.
- `using (true)`, `with check (true)` 정책을 사용자 권한 기준 정책으로 교체합니다.
- anon role의 수정/삭제 권한을 제거합니다.
- 가능하면 저장/수정/삭제는 서버 API를 통해 처리합니다.
- `created_by`, `updated_by`, 접근 로그 같은 감사 정보를 추가합니다.

### 완료 기준

- 익명 사용자는 환자 데이터를 조회, 수정, 삭제할 수 없습니다.
- 허가된 사용자만 환자 데이터를 생성하거나 조회할 수 있습니다.
- RLS 정책이 문서화되어 있고 테스트로 확인됩니다.

## 2. 데모 앱에 실제 환자 개인정보 입력 금지 안내 필요

### 문제

현재 화면은 이름, 생년월일, 연락처, 개인 ID 번호, 가족 정보 같은 개인정보를 입력받습니다.

### 위험

테스트 중 실제 환자 정보를 입력하면 브라우저 상태, 네트워크 요청, 로그, GitHub 기록, 잘못 설정된 Supabase 테이블을 통해 민감한 정보가 노출될 수 있습니다.

### 개선 방향

- 운영 전까지는 테스트/데모용 화면이라는 안내를 추가합니다.
- 인증과 RLS 정책이 강화되기 전에는 가짜 데이터만 사용합니다.
- 실제로 필요한 필드와 불필요한 필드를 다시 검토합니다.
- 개인 ID 번호와 연락처 필드에는 유효성 검사와 마스킹을 추가합니다.

### 완료 기준

- 앱 화면에서 데모/테스트 용도임을 명확히 알 수 있습니다.
- 테스트에는 실제 환자 데이터가 사용되지 않습니다.
- 민감한 필드에 유효성 검사와 마스킹 규칙이 적용됩니다.

## 3. 운영 전 Supabase 설정값을 환경변수로 분리 필요

### 문제

현재 Supabase project URL과 anon public key가 `src/app.js`와 빌드된 `dist` 파일에 기본값으로 들어 있습니다.

### 위험

anon key는 원래 브라우저에 노출되는 공개 키이지만, 데모용 RLS 정책이 열려 있는 상태에서는 저장소에 값이 남아 있는 것이 오용 가능성을 높입니다.

### 개선 방향

- `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` 같은 Vite 환경변수로 설정을 분리합니다.
- 실제 값 대신 `.env.example` 파일만 저장소에 올립니다.
- 환경변수 방식으로 바꾼 뒤 `dist` 파일을 다시 빌드합니다.

### 완료 기준

- 실제 Supabase 값이 소스 코드에 하드코딩되어 있지 않습니다.
- 새 개발자가 `.env.example`을 보고 설정할 수 있습니다.
- 환경변수 기반으로도 빌드와 실행이 정상 동작합니다.

## 4. Supabase URL 생성 로직 테스트 추가 필요

### 문제

현재 앱은 `https://project.supabase.co`와 `https://project.supabase.co/rest/v1/` 형태를 모두 처리하도록 URL을 정규화합니다. 이 로직은 중요하지만 자동 테스트가 없습니다.

### 위험

나중에 코드를 수정하다가 `/rest/v1/rest/v1`처럼 URL이 중복되는 문제가 다시 생기면 저장, 검색, 삭제 기능이 모두 깨질 수 있습니다.

### 개선 방향

- URL 정규화 로직을 작은 helper 함수로 분리합니다.
- trailing slash가 있는 경우와 없는 경우를 테스트합니다.
- `/rest/v1/`이 이미 포함된 경우도 테스트합니다.
- `package.json`에 테스트 명령을 추가합니다.

### 완료 기준

- URL 정규화 로직이 테스트로 보호됩니다.
- 로컬에서 `npm test` 또는 같은 역할의 명령으로 확인할 수 있습니다.
